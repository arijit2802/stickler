# Feature 11 — Interview Mode Podcast

## Goal
Add an "Interview" tab to the blog summary card that converts the article into a two-voice podcast — a female interviewer named Jane asks probing questions while the article's author answers in their own voice. This gives users a more engaging, conversational format alongside the existing single-narrator podcast.

---

## User Stories

- As a user, I want to listen to a blog article as a two-person interview so that I can absorb the content in a more natural, conversational format.
- As a user, I want Jane (interviewer) and the article author (interviewee) to have distinct voices so I can clearly follow who is speaking.
- As a user, I want the interview audio to generate on demand (like the existing podcast) so unused articles don't incur unnecessary TTS costs.

---

## Interaction Flow

```
1. User opens BlogSummaryCard → sees "Interview" tab alongside Summary / Keywords / Listen
2. User clicks "Interview" tab
3. If interviewAudioStatus = 'none': show "Generate interview" button
4. User clicks "Generate interview"
5. Client calls POST /api/blogs/:id/interview
6. Server marks interviewAudioStatus = 'generating'
7. Server:
   a. Fetches article content (from blog_summaries.learningShort or re-fetches rawContent)
   b. Calls Claude to write a two-voice interview script (Jane / Author)
   c. Splits script into segments by speaker
   d. Calls OpenAI TTS for each segment (Jane → shimmer, Author → onyx)
   e. Concatenates audio buffers into a single MP3
   f. Uploads to Vercel Blob as audio/{blogId}-interview.mp3
   g. Saves interviewAudioUrl + interviewScript + sets interviewAudioStatus = 'done'
8. Client polls GET /api/blogs/:id/interview every 5s until status = 'done' | 'failed'
9. Player appears — same controls as existing podcast player (play/pause, scrub, −15s, speed)
```

---

## Data Model

No new tables. Two new columns on `blog_summaries`:

```sql
-- Interview script (two-voice dialog generated by Claude)
ALTER TABLE blog_summaries ADD COLUMN interview_script text;

-- Vercel Blob URL for the concatenated interview MP3
ALTER TABLE blog_summaries ADD COLUMN interview_audio_url text;

-- none | generating | done | failed
ALTER TABLE blog_summaries ADD COLUMN interview_audio_status text DEFAULT 'none';
```

## API Routes

| Method | Route | Description |
|---|---|---|
| GET | `/api/blogs/[id]/interview` | Returns `interviewAudioUrl`, `interviewAudioStatus`, `interviewScript` |
| POST | `/api/blogs/[id]/interview` | Triggers interview generation; 409 if already generating |

## Service Layer

### `src/services/interview.ts` (new)

- **`writeInterviewScript(content, author)`** — Claude prompt that generates a structured dialog:
  - 6–8 Q&A exchanges
  - Jane asks thoughtful, curious questions; Author explains from personal experience
  - Each line prefixed with `JANE:` or `AUTHOR:` for reliable parsing
  - ~500–650 words total (≈ 3–4 min at natural speech pace)
  - If author name is unknown, Author speaks generically as the writer

- **`generateInterviewAudio(blogId)`** — Orchestration:
  1. Guard: read `interviewAudioStatus`; throw if already `generating`
  2. Set `interviewAudioStatus = 'generating'`
  3. Fetch article content via `getBlogContent(blogId)` (rawContent fallback chain)
  4. Call `writeInterviewScript(content, author)`
  5. Parse script into `{ speaker: 'JANE' | 'AUTHOR', text: string }[]` segments
  6. For each segment, call `openai.audio.speech.create({ voice: jane → 'shimmer', author → 'onyx' })`
  7. Collect `ArrayBuffer` chunks; concatenate into a single `Buffer`
  8. Upload to Vercel Blob: `audio/{blogId}-interview.mp3`
  9. Call `saveInterviewAudio(blogId, url, script)` → sets `interviewAudioStatus = 'done'`
  10. On any error: set `interviewAudioStatus = 'failed'`; rethrow

### `src/models/summarisation.ts` (extend)

- **`getInterviewStatus(blogId)`** — Returns `{ interviewAudioUrl, interviewAudioStatus, interviewScript }`
- **`saveInterviewAudio(blogId, url, script)`** — Sets url, script, status = 'done'
- **`setInterviewAudioStatus(blogId, status)`** — Used for generating / failed transitions

### `src/components/BlogSummaryCard.tsx` (extend)

- Add `"interview"` to the tab union: `"summary" | "keywords" | "short" | "interview"`
- Interview tab content:
  - `interviewAudioStatus = 'none'`: "Generate interview" button → POST /api/blogs/:id/interview
  - `interviewAudioStatus = 'generating'`: spinner + polls GET every 5s
  - `interviewAudioStatus = 'done'`: same `<audio>` player as Listen tab; below it, optional collapsible transcript
  - `interviewAudioStatus = 'failed'`: error + "Retry" button

## Voice Mapping

| Speaker | OpenAI Voice | Character |
|---|---|---|
| Jane (interviewer) | `shimmer` | Warm, curious female voice |
| Author (interviewee) | `onyx` | Deep, authoritative — same voice as the solo podcast |

> Note: OpenAI TTS voices are fixed. `shimmer` is the closest to a distinct female voice for Jane.

---

## Script Format Contract

Claude must produce each line in exactly this format (used by the parser):

```
JANE: [question or transition sentence]
AUTHOR: [answer, 2–4 sentences]
```

The parser splits on newlines, reads the prefix, and routes to the correct TTS voice. Lines that don't start with `JANE:` or `AUTHOR:` are skipped.

---

## Audio Concatenation

MP3 segments are concatenated as raw `Buffer` data (no re-encoding). This is valid for playback — browsers handle multi-segment MP3s correctly. No ffmpeg or additional binary required.

```ts
const buffers = await Promise.all(segments.map(seg => ttsCall(seg)));
const combined = Buffer.concat(buffers);
```

> Segments are generated **sequentially** (not in parallel) to preserve dialog order and avoid OpenAI rate limits.

---

## Edge Cases

| Case | Handling |
|---|---|
| Author name unknown | Script prompt instructs Claude to write Author as "the writer" without a specific name |
| Article too short (< 200 words) | Use available content; Claude may produce fewer exchanges (min 4) |
| TTS call fails mid-generation | Catch error, set `interviewAudioStatus = 'failed'`; partial audio is discarded |
| Already generating (double-click) | POST returns 409; client ignores and continues polling |
| Retry after failed | POST /api/blogs/:id/interview resets status to `generating` and restarts |
| Very long article | `content.slice(0, 8000)` cap on Claude input; same as existing podcast |
| Blob upload fails | Set status `failed`; log error; do not store partial URL |
